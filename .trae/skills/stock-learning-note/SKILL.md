# 股票学习笔记记录 Skill

## 何时调用

在以下情况下调用此 Skill：
- 用户说"我今天读了..."
- 用户分享阅读体会或笔记
- 用户说"记录学习进度"
- 用户提到具体章节内容
- 用户说"继续记录阅读笔记"

## 功能说明

这个 Skill 用于自动化股票学习笔记记录流程：

1. **读取核心记忆** - 从 `.trae/memory/memory.md` 获取项目状态
2. **读取当前状态** - 从 `01_学习计划/当前阅读状态.md` 获取阅读进度
3. **整理笔记** - 将用户的体会整理成结构化格式
4. **更新进度** - 更新当前阅读状态和书籍笔记
5. **更新记忆** - 更新 `.trae/memory/memory.md` 关键信息
6. **提交保存** - Git 提交到 GitHub
7. **记录反馈** - 记录本次执行情况用于进化优化

## 使用方法

### 标准流程

```
用户: "我今天读了《富爸爸穷爸爸》第3章，关于资产和负债的..."

AI:
1. 读取 .trae/memory/memory.md（核心记忆）
2. 读取 01_学习计划/当前阅读状态.md
3. 确认当前书籍和章节
4. 整理用户分享的笔记
5. 更新 01_学习计划/当前阅读状态.md 进度
6. 更新 02_阅读笔记/书籍笔记/01_富爸爸穷爸爸.md
7. 更新 .trae/memory/memory.md 关键信息
8. Git 提交
9. 记录执行反馈到 usage-log.md
10. 定期询问用户满意度（每5次或用户主动反馈时）
11. 根据反馈优化
```

- **核心记忆**: `.trae/memory/memory.md` ⭐ 新增
- **每日记录**: `.trae/memory/daily/YYYY-MM-DD.md` ⭐ 新增
- **当前阅读状态**: `01_学习计划/当前阅读状态.md` ✅ 已更新路径
- **学习计划总览**: `01_学习计划/学习计划总览.md` ✅ 已更新路径
- **书籍笔记**: `02_阅读笔记/书籍笔记/XX_书名.md`
- **笔记模板**: `02_阅读笔记/笔记整理模板.md`
- **使用日志**: `.trae/skills/stock-learning-note/usage-log.md`
- **进化日志**: `.trae/skills/stock-learning-note/evolution-log.md`

### 笔记格式

整理后的笔记包含：
- 日期和阅读时长
- 章节核心内容概括
- 用户的重要体会
- 经典语录摘录
- 实践任务清单
- 下一步计划

## 注意事项

1. **先读记忆文件** - 每次执行先读取 `.trae/memory/memory.md`
2. **自动识别书籍** - 从用户描述中识别书名和章节
3. **保持连续性** - 基于之前的内容继续记录
4. **结构化整理** - 使用统一的笔记格式
5. **及时提交** - 每次记录后提交到 GitHub
6. **收集反馈** - 定期询问用户满意度（每5次或用户主动反馈时）
7. **持续进化** - 根据反馈优化 Skill

## 输出示例

**用户输入**:
"我今天读了第3章，关于资产和负债的区别。最大的收获是明白了房子不一定是资产，如果它每个月让你花钱就是负债。"

**AI 整理后**:
```markdown
### 第3章学习记录 - 2026-04-06

**核心观点**:
- 资产是把钱放进你口袋的东西
- 负债是把钱从你口袋拿走的东西
- 房子不一定是资产

**我的体会**:
- 颠覆了传统观念
- 要用现金流思维看待资产

**待实践**:
- [ ] 列出自己的资产和负债清单
```

## 进化机制

### 反馈收集

定期询问用户满意度（每5次或用户主动反馈时）：
- "最近的笔记整理是否符合你的期望？"
- "有哪些需要改进的地方？"
- "你希望增加什么功能？"

### 使用记录

记录到 `usage-log.md`：
- 执行时间
- 用户输入摘要
- 执行结果
- 用户评分（1-5星）
- 用户反馈
- 改进建议

### 进化触发条件

当以下情况发生时，触发 Skill 进化：
1. 连续3次评分低于4星
2. 用户明确提出改进建议
3. 累计使用10次后自动review
4. 用户说"优化skill"或"改进笔记功能"

### 进化流程

```
1. 分析 usage-log.md 中的反馈
2. 识别问题和改进点
3. 更新 SKILL.md
4. 记录进化日志
5. 测试新版本
6. 提交到 GitHub
```
